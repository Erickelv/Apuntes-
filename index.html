SKILLS IT

Componentes de una Computadora

Una computadora está compuesta por varios componentes clave que trabajan juntos:

  • Placa base (Motherboard): El circuito principal que conecta todos los componentes.
  • CPU (Procesador): El cerebro de la computadora que ejecuta instrucciones.
  • RAM (Memoria): La memoria de trabajo a corto plazo donde se almacenan temporalmente los datos que la CPU necesita acceder rápidamente.
  • GPU (Tarjeta gráfica): Procesa y genera las imágenes que ves en pantalla. Puede ser discreta o integrada.
  • GPU Discreta: Tarjeta gráfica independiente con su propia memoria, más potente para gaming y aplicaciones gráficas intensivas.
  • GPU Integrada: Procesador gráfico incluido en la CPU, más económica pero menos potente.
  • Almacenamiento (SSD/HDD): Donde se guardan el sistema operativo, los programas y tus archivos permanentemente.
  • Fuente de poder (PSU): Suministra electricidad a todos los componentes.
  • Gabinete (Case): La caja que protege y aloja todo.

Tipos de Conexiones y Funcionalidades

  • NFC (Near Field Communication): Permite la comunicación inalámbrica a muy corta distancia (pocos centímetros), usada para pagos móviles y para emparejar dispositivos rápidamente.
  • WIFI (Estándar 802.11): Es la tecnología que permite la conexión inalámbrica a internet y redes locales (WLAN). Utiliza ondas de radio para transmitir datos.
  • Bluetooth: Una tecnología de comunicación inalámbrica de corto alcance diseñada para conectar dispositivos como audífonos, teclados, ratones y altavoces.
  • Intranet: Es una red privada que pertenece a una organización. Utiliza las mismas tecnologías que Internet, pero solo es accesible para los miembros de esa organización.

Solucionar Problemas de Sistemas Operativos

Para solucionar problemas en sistemas operativos es fundamental entender:

  • Los procesos y servicios del sistema
  • Los registros de eventos (logs)
  • La administración de usuarios y permisos
  • La configuración de red
  • Los controladores de dispositivos
  • Las herramientas de diagnóstico integradas

Suites Populares

Una Suite de Software es un conjunto de programas informáticos relacionados que se venden o distribuyen como un solo paquete. Suelen tener una interfaz de usuario similar y están diseñados para trabajar juntos de manera fluida.

Microsoft Office Suite (Ahora Microsoft 365)

Es la suite de productividad más conocida, enfocada en entornos de escritorio y empresariales. Incluye Word, Excel, PowerPoint, Outlook, OneNote, Teams, etc.

Google Suite (Ahora Google Workspace)

Es una suite basada en la nube, enfocada en la colaboración en tiempo real. Incluye Docs, Sheets, Slides, Gmail, Drive, Calendar, etc.

Administración de Cuentas de Dominio

Saber administrar cuentas de dominio implica gestionar usuarios, grupos, políticas de seguridad y permisos en un entorno de red corporativa, generalmente usando Active Directory en Windows Server.

Configuración de Correos y Protocolos

Configurar un cliente de correo (como Outlook) implica usar protocolos específicos:

  • IMAP (Internet Message Access Protocol): Sincroniza tu correo con el servidor. Es el más recomendado.
  • POP3 (Post Office Protocol 3): Descarga los correos del servidor a tu PC, usualmente borrándolos del servidor.
  • SMTP (Simple Mail Transfer Protocol): Es el protocolo estándar que se usa exclusivamente para enviar correos electrónicos.

Sistemas Operativos

Instalación y Configuración

Entrar a la BIOS/UEFI: La BIOS/UEFI es el firmware de la placa base, el primer software que se ejecuta al encender un PC. Para entrar, debes presionar una tecla específica justo al arrancar (comúnmente F2, F10, F12 o DEL).

Instalar un Sistema Operativo Linux

Pasos básicos: crear medio de instalación (USB), configurar BIOS/UEFI para arrancar desde USB, seguir el asistente de instalación, particionar el disco, configurar usuario y contraseña.

Instalar un Sistema Operativo Windows

Similar a Linux: crear medio de instalación, arrancar desde él, seguir el asistente, particionar el disco, configurar usuario y preferencias.

Versiones, Diferencias y Propósitos

Windows

  • Windows Home: Para usuarios domésticos.
  • Windows Pro: Para profesionales. Añade funciones como BitLocker (cifrado de disco), Escritorio Remoto y Políticas de Grupo.
  • Compatibilidad: Excelente compatibilidad con software y hardware, especialmente para gaming y aplicaciones comerciales.

Linux

  • Ubuntu/Mint: Ideales para principiantes. Propósito: Escritorio, desarrollo.
  • CentOS/RHEL/Debian: Estándar en servidores por su estabilidad. Propósito: Servidores, infraestructura.
  • Arch Linux: Para usuarios avanzados que quieren construir su sistema desde cero. Propósito: Aprendizaje, personalización.
  • Compatibilidad: Perfecta para desarrollo y ciberseguridad.

Permisos y Privilegios en Servidores

Los sistemas operativos utilizan un modelo de privilegios para protegerse.

  • Linux: El superusuario es root. Los usuarios normales usan el comando sudo para ejecutar tareas con privilegios elevados temporalmente. Los archivos tienen permisos de Lectura (r), Escritura (w) y Ejecución (x) para un Dueño, un Grupo y los Otros.
  • Windows: El superusuario es el grupo de Administradores. Se utiliza el Control de Cuentas de Usuario (UAC) que solicita confirmación antes de realizar tareas importantes. Los permisos se gestionan con Listas de Control de Acceso (ACLs).

Permisos de Ejecución en PowerShell

PowerShell restringe la ejecución de scripts por seguridad. Es crucial saber cómo gestionarla.

# Muestra las políticas de ejecución para todos los ámbitos (Scopes)
Get-ExecutionPolicy -List

El Scope (ámbito) en Set-ExecutionPolicy determina qué tan permanente es el cambio:

  • -Scope Process: El cambio solo dura para la ventana actual de la terminal. Es la opción más segura y temporal.
  • -Scope CurrentUser: El cambio es persistente para tu usuario y se guarda para futuras sesiones.
# Permite ejecutar scripts firmados, solo en esta sesión (Temporal y seguro)
Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned

Anillos de Privilegio (Protection Rings)

Los sistemas operativos modernos separan los programas en niveles jerárquicos llamados anillos para proteger el sistema. Un proceso en un anillo de menor número tiene más privilegios.

  • Anillo 0 (Kernel Mode): El nivel más privilegiado. Aquí reside el Kernel, el núcleo del sistema operativo. Tiene acceso directo y sin restricciones a todo el hardware. Un fallo aquí es catastrófico y causa un "pantallazo azul" o "kernel panic".
  • Anillo 1 y 2 (Driver Mode): Niveles intermedios. Aquí operan los controladores de dispositivos (drivers). Tienen un acceso privilegiado al hardware, pero más restringido que el kernel. La mayoría de los sistemas operativos modernos no utilizan estos anillos de forma explícita, integrando los drivers en el Anillo 0 o 3.
  • Anillo 3 (User Mode): El nivel menos privilegiado. Aquí se ejecutan todas tus aplicaciones: navegadores web, juegos, editores de texto, etc. Si una aplicación falla, solo se cierra ella misma sin afectar al resto del sistema operativo. No pueden acceder al hardware directamente; deben solicitarlo al kernel.

Redes

Modelo OSI vs. TCP/IP

Modelo OSI: Es un modelo teórico de 7 capas creado para estandarizar la comunicación en redes. Es perfecto para aprender y entender cómo viaja la información, pero no es el que se usa en la práctica.

Modelo TCP/IP: Es un modelo práctico de 4/5 capas sobre el que está construido todo el internet. Es la implementación real de los conceptos del modelo OSI.

Protocolos Fundamentales

Protocolo Definición y Uso
SSH (Secure Shell) - Protocolo para conectarse de forma segura a la terminal de un servidor remoto.
RDP (Remote Desktop Protocol) - Permite conectarse a un computador remoto con interfaz gráfica.
FTP (File Transfer Protocol) - Sirve para subir y descargar archivos, pero no es seguro.
SFTP/FTPS (Secure FTP) - Versiones seguras del FTP que encriptan la conexión.
HTTP (Hypertext Transfer Protocol) - Envía información web sin encriptar.
HTTPS (HTTP Secure) - La versión segura de HTTP. Envía información encriptada.

Conexiones de Red y Protección de Puertos

Una conexión de red se establece cuando un cliente solicita un servicio a un servidor en un puerto específico. Los puertos son como las puertas de un edificio: algunos deben estar abiertos al público, pero muchos deben estar cerrados y protegidos.

Puertos Comúnmente Abiertos: 80 (HTTP), 443 (HTTPS), 22 (SSH), 53 (DNS), 25 (SMTP).

¿Cómo Protegerlos?

  • Firewall: Es la primera línea de defensa. Actúa como un guardia de seguridad que permite o bloquea el tráfico según reglas predefinidas. Configúralo para permitir solo las conexiones necesarias.
  • Principio de Menor Privilegio: No abras un puerto si no es absolutamente necesario. Cierra todo por defecto y abre solo lo indispensable.
  • Hardening de Servicios: Asegura el software que se ejecuta en los puertos abiertos. Por ejemplo, para SSH (puerto 22), deshabilita el login de root y usa autenticación con llaves en lugar de contraseñas.
  • Monitoreo y Escaneo: Usa herramientas como Nmap para escanear tus propios puertos y ver qué tienes abierto. Utiliza sistemas de detección de intrusos (IDS) para monitorear el tráfico en busca de actividad sospechosa.

Almacenamiento en Red: NAS y SAN

  • NAS (Network Attached Storage): Es un dispositivo de almacenamiento conectado a la red que funciona a nivel de archivos. Es como tener un disco duro compartido, ideal para centralizar archivos en una casa u oficina pequeña.
  • SAN (Storage Area Network): Es una red de alta velocidad dedicada al almacenamiento que funciona a nivel de bloques. Es de alto rendimiento y se usa en entornos empresariales para bases de datos y virtualización.

SSL y TLS

SSL (Secure Sockets Layer): Protocolo original para encriptar. Está obsoleto.

TLS (Transport Layer Security): Es el sucesor moderno y seguro de SSL, usado para HTTPS.

Conceptos de Red Específicos

Subnetting

Es la práctica de dividir una red grande en redes más pequeñas llamadas "subredes". Permite una mejor organización, seguridad y rendimiento de la red.

Terminología IP Clave

  • IP Privada vs. IP Pública: La IP privada es para tu red local interna; la pública es para identificarte en internet.
  • localhost: Nombre que siempre se refiere a tu propia máquina (127.0.0.1).
  • Dirección de Loopback: La dirección IP 127.0.0.1 que siempre apunta al propio dispositivo.
  • Subnetting: Es la práctica de dividir una red grande en redes más pequeñas llamadas "subredes".
  • Máscara de Subred (Subnet Mask): Un número (ej. 255.255.255.0) que le dice a un dispositivo qué parte de una dirección IP es la red y qué parte es el host.
  • CIDR (Classless Inter-Domain Routing): Notación moderna para una IP y su máscara de red (ej. 192.168.1.0/24).
  • Default Gateway: La dirección IP de tu router. Es la "puerta de salida" de tu red local.

Terminología de Red

  • VLAN (Virtual LAN): Permite crear múltiples redes lógicas y aisladas sobre una misma infraestructura física.
  • DMZ (Demilitarized Zone): Una subred intermedia entre tu red interna e internet para alojar servidores públicos.
  • ARP (Address Resolution Protocol): Traduce direcciones IP a direcciones MAC en una red local.
  • VM (Virtual Machine): Máquina virtual, un entorno de sistema operativo emulado.
  • DHCP (Dynamic Host Configuration Protocol): Asigna automáticamente direcciones IP a los dispositivos.
  • DNS (Domain Name System): El "directorio de internet" que traduce nombres (google.com) a direcciones IP.
  • NAT (Network Address Translation): Permite que múltiples dispositivos compartan una única IP pública.
  • IP (Internet Protocol): Protocolo fundamental para el direccionamiento y enrutamiento de paquetes.
  • Router: Dispositivo que conecta redes diferentes y opera en la capa 3 del modelo OSI.
  • Switch: Dispositivo que conecta dispositivos en una misma red y opera en la capa 2 del modelo OSI.
  • VPN (Virtual Private Network): Crea una conexión segura y encriptada sobre una red pública.

Tipos de Redes

  • LAN (Local Area Network): Red de área local (una casa, una oficina).
  • WAN (Wide Area Network): Red de área amplia que conecta redes a través de grandes distancias geográficas (Internet es una WAN).
  • WLAN (Wireless Local Area Network): Una red local inalámbrica (tu red Wi-Fi).
  • MAN (Metropolitan Area Network): Una red que cubre un área geográfica como una ciudad.

Protocolos de Servicios de Red

  • DHCP (Dynamic Host Configuration Protocol): Asigna automáticamente direcciones IP a los dispositivos.
  • DNS (Domain Name System): Traduce nombres de dominio a direcciones IP.
  • NTP (Network Time Protocol): Sincroniza la hora entre dispositivos, crucial para auditorías y logs.
